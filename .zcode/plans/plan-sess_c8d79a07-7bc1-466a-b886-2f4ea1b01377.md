# Rencana Implementasi: TorentChat — Aplikasi Chat P2P Terenkripsi

## 📋 Ringkasan Visi
Aplikasi chat Android yang mengutamakan privasi & keamanan: pesan **peer-to-peer (P2P)**, **terenkripsi end-to-end** dengan Signal Protocol, **tanpa server pusat penyimpan data**, identitas **anonymous** (ID acak + QR/link undangan), dan pesan offline tersimpan di **KV cache terenkripsi** (Cloudflare Workers edge, bukan database permanen).

---

## 🏗️ Arsitektur

```
┌─────────────────┐         ┌──────────────────┐         ┌─────────────────┐
│   Perangkat A    │         │  Cloudflare      │         │   Perangkat B    │
│  (Android App)   │         │  Workers + KV    │         │  (Android App)   │
│                  │  Signal │  (edge,          │  Signal │                  │
│  Signal Protocol │ ◄─────► │   serverless)    │ ◄─────► │  Signal Protocol │
│  WebRTC P2P      │         │                  │         │  WebRTC P2P      │
│  SQLCipher DB    │         │  • Signaling     │         │  SQLCipher DB    │
│                  │  P2P    │  • KV cache E2E  │  P2P    │                  │
│  ┌────────────┐  │ ◄═════════════════════════════════► │  ┌────────────┐  │
│  │E2EE Engine │  │    WebRTC Data Channel (langsung)   │  │E2EE Engine │  │
│  └────────────┘  │                                     │  └────────────┘  │
└─────────────────┘                                       └─────────────────┘
```

**Prinsip kunci:** Cloudflare Workers **hanya** bertindak sebagai signaling (membantu peer saling temukan) + cache ephemeral. Workers **tidak pernah** melihat isi pesan plain-text — semua yang lewat sudah terenkripsi E2E di perangkat.

## 🛠️ Tech Stack

| Komponen | Teknologi |
|---|---|
| Android App | Kotlin + Jetpack Compose + MVVM |
| Local DB terenkripsi | SQLCipher (via `net.zetetic:android-database-sqlcipher`) |
| E2EE | Signal Protocol — **libsignal** (`org.signal:libsignal`) Double Ratchet + X3DH |
| P2P transport | WebRTC (`io.getstream:stream-webrtc-android` — build modern Google WebRTC) |
| Signaling/Relay | Cloudflare Workers (TypeScript) + Workers KV |
| QR Code | `com.google.zxing:core` + `journeyapps:zxing-android-embedded` |
| DI | Hilt (`com.google.dagger:hilt-android`) |
| Async | Kotlin Coroutines + Flow |
| Networking (signaling) | Ktor Client / OkHttp + WebSocket |

## 📦 Implementation Phases

Karena lingkup sangat besar, saya akan implementasi secara **bertahap** dalam sesi ini. Saya mulai dari fondasi yang paling kritis:

### **Phase 1 — Setup & Fondasi** (akan dikerjakan sekarang)
1. Buat struktur proyek lengkap (folder + file skeleton)
2. `android-app`: Gradle config, version catalog, manifest, theme, Application class, Hilt setup
3. `worker`: Cloudflare Worker dengan endpoint signaling + KV cache (TypeScript, siap deploy)
4. Dokumentasi arsitektur (`ARCHITECTURE.md`, `SECURITY.md`, `DEPLOY.md`)
5. README dengan instruksi setup

### **Phase 2 — Identitas & Keamanan**
6. `IdentityManager`: generate ID acak + Signal identity keys
7. `SignalSessionManager`: wrapper X3DH key agreement + Double Ratchet
8. QR code generator + invite link (parse/buat)
9. Onboarding screen (generate ID pertama kali, tampilkan QR sendiri)

### **Phase 3 — Konektivitas P2P**
10. `SignalingClient`: connect ke Worker, exchange SDP/ICE
11. `WebRtcManager`: PeerConnection, DataChannel, NAT traversal
12. `DataChannelTransport`: kirim/terima pesan terenkripsi via P2P
13. `KVPendingStore`: ambil pesan offline terenkripsi dari KV

### **Phase 4 — UI & Fitur Chat**
14. Conversations list screen
15. Chat screen (bubble, input bar, kirim/terima pesan real-time)
16. Group chat (multi-peer)
17. Chunked media sender (gambar E2E)
18. Presence (online/typing indicator)

### **Phase 5 — Local Storage & Polish**
19. SQLCipher encrypted database + DAOs
20. Message history persistence
21. Settings screen (auto-delete, security options)
22. Testing & integrasi
